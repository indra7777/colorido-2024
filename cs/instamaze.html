Instamaze: A Programming Contest

Rendering as sheer intelligence, this event aims at bringing out the best programming talents among the participants.

Guidelines for the Team:

  1. Maximum number of members allowed per team is 2. It must be the same team that appeared in prelims.
  2. The maximum time allotted is 3 hours FOR Second round Main Exam.
  3. One computer and one USER ID shall be allotted to each team.
  4. The team should use only their allotted USER ID. Using other USER ID's shall lead to disqualification of the team summarily.
  5. The problems may be solved using any of the programming languages C, C++, JAVA.
  6. Non-standard libraries cannot be used in your solutions. The Standard Template Library (STL) and C++ string libraries are allowed. The standard Java API is available, except for those packages that are deemed dangerous by contest officials (e.g., that might generate a security violation).
  7. After successful execution of each program, the team should inform it to the volunteer to record the completion time.
  8. A team is not allowed to discuss with any other team during the contest.
  9. Any sort of indiscipline during the contest shall lead to cancellation of participation of the team.
  10. All programs will be re-compiled prior to testing with the judge's data.
  11. The team that solves the maximum number of the given problems with the best logic possible and appropriate programming style will be the winner.
  12. In case of a tie, logic, and the simplicity of the code are the criteria for finalizing the winner.
  13. The decision of the judge is final.
